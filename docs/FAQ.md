# Cherry Note 常见问题解答 (FAQ)

## 安装和设置

### Q: 支持哪些操作系统？
**A:** Cherry Note 支持以下操作系统：
- Android 5.0 (API 21) 及以上版本
- Windows 10 及以上版本
- macOS 10.14 (Mojave) 及以上版本

### Q: 如何获取S3存储配置信息？
**A:** 您需要从S3兼容的云存储服务提供商获取以下信息：
- **端点URL**: 如 `https://s3.amazonaws.com` 或自建服务的URL
- **Access Key ID**: 访问密钥ID
- **Secret Access Key**: 密钥
- **存储桶名称**: 用于存储笔记的存储桶

常见的S3兼容服务：
- Amazon S3
- 阿里云OSS
- 腾讯云COS
- MinIO (自建)
- Backblaze B2

### Q: 首次安装后无法连接S3怎么办？
**A:** 请检查以下几点：
1. 网络连接是否正常
2. S3配置信息是否正确
3. 存储桶是否存在且有读写权限
4. 防火墙是否阻止了应用访问网络
5. 如果使用自建服务，确认SSL证书是否有效

## 使用功能

### Q: 如何导入现有的Markdown文件？
**A:** 有两种方式导入：
1. **文件导入**: 文件菜单 → 导入 → 选择Markdown文件
2. **文件夹导入**: 文件菜单 → 导入文件夹 → 选择包含Markdown文件的文件夹

导入时会保持原有的文件夹结构。

### Q: 支持哪些Markdown语法？
**A:** Cherry Note 支持标准Markdown语法和部分扩展语法：

**基础语法:**
- 标题 (H1-H6)
- 段落和换行
- 强调 (粗体、斜体)
- 列表 (有序、无序)
- 链接和图片
- 代码 (行内和代码块)
- 引用
- 水平分割线

**扩展语法:**
- 表格
- 删除线
- 任务列表
- 语法高亮代码块

### Q: 如何插入图片？
**A:** 有三种方式插入图片：
1. **拖拽**: 直接将图片文件拖拽到编辑器
2. **工具栏**: 点击图片按钮选择本地图片
3. **Markdown语法**: `![描述](图片URL)`

本地图片会自动上传到S3存储。

### Q: 标签功能如何使用？
**A:** 标签使用方法：
1. **添加标签**: 在编辑器顶部的标签输入框输入标签名，按回车添加
2. **删除标签**: 点击标签旁的 × 按钮
3. **过滤笔记**: 在左侧标签区域点击标签进行过滤
4. **多标签过滤**: 可以选择多个标签，支持AND/OR逻辑

## 同步和数据

### Q: 数据如何在多设备间同步？
**A:** 同步机制：
1. **自动同步**: 应用会定期自动同步数据到S3
2. **实时同步**: 编辑笔记后会立即同步
3. **启动同步**: 应用启动时会检查并同步更新
4. **手动同步**: 可以在设置中手动触发同步

### Q: 离线时能否使用应用？
**A:** 是的，Cherry Note 支持离线使用：
- 已下载的笔记可以离线查看和编辑
- 离线时的更改会保存在本地
- 恢复网络连接后会自动同步所有更改
- 离线状态会在界面上显示

### Q: 如何处理同步冲突？
**A:** 当检测到冲突时，应用会提供以下选项：
1. **保留本地版本**: 使用本地修改覆盖远程版本
2. **保留远程版本**: 使用远程版本覆盖本地修改
3. **创建副本**: 保留两个版本，添加时间戳区分
4. **手动合并**: 显示差异，让用户手动选择保留的内容

### Q: 如何备份我的笔记数据？
**A:** 多种备份方式：
1. **导出备份**: 文件菜单 → 导出 → 选择ZIP格式
2. **S3备份**: 数据已存储在S3中，具有高可靠性
3. **本地缓存**: 应用本地保留数据缓存
4. **定期导出**: 建议定期导出重要笔记作为额外备份

## 性能和故障排除

### Q: 应用启动很慢怎么办？
**A:** 可能的解决方案：
1. **清理缓存**: 设置 → 清理本地缓存
2. **检查网络**: 确保网络连接稳定
3. **减少笔记数量**: 归档不常用的笔记
4. **重启应用**: 完全关闭应用后重新启动

### Q: 编辑器卡顿怎么解决？
**A:** 优化建议：
1. **分割长文档**: 将超长笔记分割为多个文件
2. **关闭实时预览**: 编辑长文档时可临时关闭预览
3. **调整字体大小**: 较小的字体可能提升性能
4. **清理内存**: 定期重启应用释放内存

### Q: 同步失败怎么办？
**A:** 故障排除步骤：
1. **检查网络连接**
2. **验证S3配置**: 设置 → S3配置 → 测试连接
3. **检查存储空间**: 确保S3存储桶有足够空间
4. **查看错误日志**: 设置 → 关于 → 查看日志
5. **重新配置**: 删除配置后重新设置

### Q: 如何查看应用日志？
**A:** 查看日志方法：
1. **应用内查看**: 设置 → 关于 → 查看日志
2. **导出日志**: 设置 → 关于 → 导出日志文件
3. **日志位置**:
   - Windows: `%APPDATA%/CherryNote/logs/`
   - macOS: `~/Library/Application Support/CherryNote/logs/`
   - Android: 应用私有目录

## 高级功能

### Q: 如何自定义主题？
**A:** 主题自定义：
1. **内置主题**: 设置 → 外观 → 选择浅色/深色主题
2. **字体设置**: 设置 → 外观 → 调整字体大小和字体族
3. **编辑器主题**: 设置 → 编辑器 → 选择语法高亮主题

### Q: 支持插件或扩展吗？
**A:** 当前版本不支持第三方插件，但我们计划在未来版本中添加插件系统。

### Q: 如何批量操作笔记？
**A:** 批量操作功能：
1. **批量导出**: 选择文件夹后导出整个文件夹
2. **批量标签**: 选择多个笔记后批量添加/删除标签
3. **批量移动**: 拖拽选择多个笔记到目标文件夹
4. **批量删除**: 选择多个笔记后批量删除

## 技术支持

### Q: 如何报告Bug或提出功能建议？
**A:** 联系方式：
1. **GitHub Issues**: 在项目GitHub页面提交Issue
2. **邮件支持**: support@cherrynote.app
3. **用户社区**: 加入我们的用户交流群

### Q: 应用是开源的吗？
**A:** 是的，Cherry Note 是开源项目，您可以在GitHub上查看源代码、提交Issue或贡献代码。

### Q: 如何获取更新？
**A:** 更新方式：
- **Android**: 通过应用商店自动更新
- **Windows/macOS**: 应用内会提示更新，或从官网下载最新版本
- **开发版本**: 可以从GitHub Releases页面下载最新开发版本

### Q: 数据隐私和安全如何保障？
**A:** 隐私保护措施：
1. **本地加密**: 敏感配置信息在本地加密存储
2. **传输加密**: 所有网络传输使用HTTPS/TLS加密
3. **无数据收集**: 应用不收集用户个人信息
4. **开源透明**: 代码完全开源，可审计安全性
